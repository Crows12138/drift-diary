# AI日记漂流瓶 - 产品需求文档（PRD）

> 版本：v1.0 | 最后更新：2026-03-14

---

## 一、产品概述

### 1.1 产品名称
**漂流日记**（暂定，待确定）

### 1.2 产品定位
一款面向年轻人的日记应用，让用户安全地表达自我、被AI理解、与陌生人产生温暖连接。

### 1.3 一句话描述
> 写下你的故事，AI读懂你的心情，让日记漂向远方，带回陌生人的温暖。

### 1.4 目标用户
- **年龄**：16-28岁
- **画像**：有表达欲但缺少出口的年轻人；渴望被理解又不想暴露身份；对陌生人社交好奇但社交压力大
- **使用场景**：深夜独处、工作间歇、情绪波动时

### 1.5 核心价值
| 维度 | 价值 |
|------|------|
| 功能价值 | 写日记 + AI分析 + 匿名社交 |
| 情感价值 | 被理解、不孤单、有惊喜 |
| 社会价值 | 陌生人之间的善意传递 |

## 二、核心功能

### 2.1 写日记

#### 2.1.1 基本功能
- 纯文字日记，支持长文书写
- 每篇日记可选择**心情标签**（如：开心、平静、焦虑、难过、愤怒、迷茫、感恩等）
- 支持添加**主题标签**（如：#工作 #恋爱 #成长 #日常 等）
- 日记按时间线排列，支持日历视图浏览
- 支持搜索历史日记

#### 2.1.2 写作模式
- **自由书写**：空白页面，自由表达
- **AI引导书写**：AI 每天提供写作提示/问题，用户回答即为日记（降低写作门槛）
  - 示例引导语：
    - "今天让你笑的一件小事是什么？"
    - "如果今天的心情是一首歌，会是哪首？"
    - "现在最想对谁说一句话？"

#### 2.1.3 日记属性
每篇日记有两种属性，由用户写完后选择：
- **私密日记** 🔒：仅自己可见，不参与漂流
- **漂流日记** 🍾：进入漂流池，等待被陌生人捡到

---

### 2.2 AI 分析

AI 贯穿整个产品体验，扮演三层角色：

#### 2.2.1 镜子 — 情绪洞察
- 每篇日记写完后，AI 自动识别情绪并给出简短反馈
- 示例："你今天提到了3次'压力'，看起来最近有点累。要不要聊聊？"
- 情绪识别维度：快乐、悲伤、焦虑、愤怒、平静、感恩、迷茫、兴奋

#### 2.2.2 日记本 — 成长记录
- **情绪曲线**：以周/月/年为维度展示情绪变化趋势
- **月度报告**：每月自动生成情绪总结报告
  - 这个月出现最多的情绪
  - 情绪最高和最低的日子
  - 关键词词云
  - 一段 AI 总结性点评
- **年度回顾**：年底生成年度情绪旅程回顾

#### 2.2.3 朋友 — 对话式陪伴
- 用户写完日记后，可以选择与 AI 对话
- AI 基于当天日记内容，像朋友一样给出回应
  - 不是冷冰冰的分析，而是温暖、共情的对话
  - 可以追问、给建议、鼓励、或只是倾听
- AI 记住用户的历史日记，对话有连续性
  - 示例："上周你说要去面试，怎么样了？"

#### 2.2.4 红娘 — 智能匹配（服务于漂流功能）
- AI 分析漂流日记的内容、情绪、主题
- 根据匹配策略选择合适的读者（详见 2.3 节）

---

### 2.3 漂流瓶

#### 2.3.1 核心机制
```
用户写日记 → 选择"漂流" → AI分析内容 → 匹配合适的读者
→ 读者阅读日记 → 写一段回应（只能添加，不能删改原文）
→ 日记带着回应继续漂流 → 下一个读者看到原文+所有回应
→ 继续添加回应 → 无限漂流下去...
```

#### 2.3.2 匹配策略
AI 根据日记内容自动选择匹配方式：
- **共鸣匹配**：相似经历/情绪的人（"我也是"的温暖）
  - 适用于：分享型、记录型日记
- **互补匹配**：把困境中的人匹配给走出来的人（"过来人"的力量）
  - 适用于：倾诉型、求助型日记

#### 2.3.3 漂流规则
- 日记一旦选择漂流，**无限漂流**，不会自动停止
- 原作者可以随时查看日记的**漂流旅程**（当前漂到了第几站、每一站的回应）
- 每个读者只能添加**一段文字回应**（纯文字）
- 回应只能**增加**，不能删除或修改已有内容（包括自己写的）
- 每个人每天可捡到的漂流瓶数量有限（防止刷量，保持珍贵感）

#### 2.3.4 漂流瓶的隐私设计
- 漂流日记中**不显示**任何用户身份信息
- 完全匿名：无昵称、无头像、无性别
- 只有内容本身在流动，人是隐形的

#### 2.3.5 漂流旅程可视化
原作者查看自己漂流日记时，看到的是一条**旅程时间线**：

```
🍾 我的漂流日记 #0312
"今天面试失败了，觉得自己什么都做不好..."

  📍 第1站 · 3月13日
  💬 "我去年也是，投了200份简历。现在在一家很喜欢的公司。会好的。"

  📍 第2站 · 3月14日
  💬 "面试失败不是你的问题，是匹配度的问题。就像这个漂流瓶，总会遇到对的人捡起来。"

  📍 第3站 · 3月14日
  💬 "抱抱你！我今天也被拒了，但看到前面的留言突然觉得没那么孤单了"

  🌊 正在继续漂流中...
```

### 2.4 情绪响应式界面

这是本产品的视觉差异化特色，**当前市场上无竞品实现此功能**。

#### 2.4.1 核心概念
App 的背景色调、氛围光会根据用户当天记录的心情自动变化：

| 心情 | 主色调 | 氛围描述 |
|------|--------|---------|
| 开心 | 暖阳橙 / 晴空蓝 | 明亮温暖，像晴天 |
| 平静 | 薄荷绿 / 月光白 | 清澈宁静，像湖面 |
| 焦虑 | 琥珀橙 / 暗金 | 微微发热，像傍晚 |
| 难过 | 紫罗兰 / 深蓝 | 安静深沉，像夜海 |
| 愤怒 | 暗红 / 深灰 | 低沉有力，像雷云 |
| 迷茫 | 雾灰 / 淡紫 | 朦胧不清，像晨雾 |
| 感恩 | 蜜桃粉 / 暖金 | 柔和温暖，像夕阳 |

#### 2.4.2 实现方式
- 默认主题为**深海蓝**（深色模式，适合深夜场景）
- 用户选择心情标签后，界面背景渐变色微调
- 变化是**细微渐进**的，不是突兀的换皮
- 用户也可以在设置中关闭此功能，使用固定主题

### 2.5 内容安全

#### 2.5.1 AI 审核
- 所有选择漂流的日记在投出前经过 AI 内容审核
- 过滤：色情/暴力/极端言论/广告/个人信息泄露
- 涉及心理危机关键词（自杀、自残等）时触发**关怀提示**，提供专业心理援助信息

#### 2.5.2 用户举报
- 读者可以举报不当的漂流日记或回应
- 举报后该内容进入人工审核队列
- 多次被举报的用户限制漂流权限

#### 2.5.3 回应审核
- 漂流瓶的回应同样经过 AI 审核
- 确保"只加不减"的善意基调

## 三、用户体验流程

### 3.1 新用户首次体验

```
下载/打开 App
  → 欢迎页（3页引导，展示核心价值）
  → 选择心情开始（不强制注册）
  → 写第一篇日记（AI引导模式，提供写作提示）
  → AI 给出第一次情绪反馈
  → 引导选择：把这篇日记放进漂流瓶？
  → 注册/登录（手机号/微信，极简流程）
  → 进入首页
```

### 3.2 日常使用流程

```
打开 App
  → 首页显示今天日期 + 情绪氛围背景
  → 底部：写日记 按钮（核心入口）
  → 写完日记 → AI分析 → 选择私密/漂流
  → 如选漂流 → 投瓶动画 → 完成

捡漂流瓶：
  → 首页/专区 → "捡一个漂流瓶" 按钮
  → 展示一篇漂流日记（含已有回应链）
  → 阅读 → 写回应 → 放回漂流
```

### 3.3 核心页面结构

```
┌──────────────────────────────┐
│        底部导航栏              │
├──────┬──────┬──────┬──────────┤
│ 首页  │ 漂流  │ AI   │  我的    │
│ 🏠   │ 🍾   │ 🤖   │  👤     │
└──────┴──────┴──────┴──────────┘
```

#### 首页
- 今日日期 + 情绪氛围背景
- 快速写日记入口
- 今日 AI 写作引导提示
- 最近日记列表

#### 漂流
- "捡一个漂流瓶" 按钮
- 我投出的漂流瓶列表（可查看旅程）
- 漂流瓶状态：漂流中 / 有新回应

#### AI
- 与 AI 对话（基于日记历史）
- 情绪曲线图表
- 月度/年度报告入口
- AI 洞察提醒

#### 我的
- 日记日历视图
- 所有日记列表（私密 + 漂流）
- 日记搜索
- 设置（主题、通知、隐私、账号）

## 四、非功能需求

### 4.1 性能要求
- 日记写作页面加载时间 < 1秒
- AI 情绪分析响应时间 < 3秒
- 漂流瓶匹配时间 < 5秒

### 4.2 安全要求
- 日记内容端到端加密存储
- 私密日记绝对不可被其他用户或系统管理员查看
- 符合《个人信息保护法》《数据安全法》要求
- 漂流日记仅在用户主动选择后才进入公共池

### 4.3 隐私声明
- 明确告知用户 AI 分析使用的数据范围
- 用户可随时删除自己的数据（包括漂流中的日记）
- 不将用户数据用于广告或第三方共享

## 五、技术方案

### 5.1 技术架构（推荐）

#### 前端
| 阶段 | 方案 | 理由 |
|------|------|------|
| MVP 验证 | 微信小程序 | 零安装、传播快、2-3周可上线 |
| 国内正式版 | UniApp (Vue3) | 一套代码出小程序 + Android + iOS |
| 国际版 | 评估 Flutter/RN 或继续 UniApp | 出海时根据需求决定 |

#### 后端
- **框架**：Node.js (Express/Koa) 或 Python (FastAPI)
- **数据库**：PostgreSQL（日记主数据）+ Redis（缓存、匹配队列）
- **对象存储**：阿里云 OSS（未来图片等资源）
- **消息队列**：RabbitMQ 或 Redis Stream（漂流瓶匹配异步处理）

#### AI 服务
- **国内**：通义千问 / DeepSeek API（情绪分析、对话、内容审核）
- **国际版**：Claude API（Anthropic）
- **NLP 能力**：情绪识别、关键词提取、内容安全检测、对话生成

#### 云服务
- **国内**：阿里云 / 腾讯云
- **国际**：AWS / GCP

### 5.2 数据模型（核心）

```
User（用户）
├── id
├── phone / wechat_openid
├── created_at
└── settings (JSON)

Diary（日记）
├── id
├── user_id
├── content (加密文本)
├── mood_tag (心情标签)
├── topic_tags (主题标签数组)
├── type: "private" | "drift"
├── ai_analysis (JSON: 情绪分析结果)
├── created_at
└── updated_at

DriftBottle（漂流瓶）
├── id
├── diary_id
├── status: "drifting" | "picked" | "removed"

├── created_at
└── matched_at

DriftResponse（漂流回应）
├── id
├── drift_bottle_id
├── user_id (匿名，仅系统内部关联)
├── content (回应文字)
├── station_number (第几站)
├── created_at

AIConversation（AI对话）
├── id
├── user_id
├── messages (JSON数组)
├── related_diary_ids
├── created_at

Report（举报）
├── id
├── reporter_user_id
├── target_type: "diary" | "response"
├── target_id
├── reason
├── status: "pending" | "reviewed" | "resolved"
├── created_at
```

## 六、MVP 范围定义

### 第一期 MVP（目标：4-6周）

**必须做的**：
- [x] 用户注册/登录（手机号 或 微信）
- [x] 写日记（纯文字 + 心情标签）
- [x] 日记列表（时间线视图）
- [x] AI 情绪分析（写完后自动分析 + 简短反馈）
- [x] 漂流瓶投出（选择漂流 → 进入匹配池）
- [x] 捡漂流瓶（随机/AI匹配 → 阅读 → 写回应 → 继续漂流）
- [x] 漂流旅程查看（原作者查看回应链）
- [x] AI 内容审核（漂流前自动审核）
- [x] 基础举报功能

**不做的（后续迭代）**：
- ❌ AI 引导书写
- ❌ AI 对话陪伴
- ❌ 情绪曲线/月度报告/年度回顾
- ❌ 情绪响应式界面
- ❌ 日历视图
- ❌ 日记搜索

### 第二期（MVP验证后）
- AI 对话陪伴
- 情绪曲线 + 月度报告
- 情绪响应式界面
- AI 引导书写
- 日历视图 + 搜索

### 第三期
- 年度回顾
- 国际版适配
- 更精细的匹配算法
- 漂流瓶统计数据（共漂流了多远等趣味数据）

## 八、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 内容安全事件 | 高 | AI审核 + 举报 + 敏感词过滤 + 危机关怀提示 |
| 用户量不足导致匹配慢 | 中 | 初期可用随机匹配兜底，降低匹配精度要求 |
| AI 分析质量不稳定 | 中 | 使用成熟的大模型 API，做好 prompt 工程 |
| 漂流日记回应质量低 | 中 | AI 审核回应质量，过滤无意义内容 |
| 隐私泄露 | 高 | 端到端加密 + 完全匿名 + 合规审查 |
